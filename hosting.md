# Hosting на GitHub Pages

## Как это работает

GitHub Pages — это бесплатный хостинг от GitHub. Он берёт файлы из твоего репозитория и раздаёт их как обычный сайт по HTTPS.

Никакого сервера, никаких баз данных — просто статические файлы (HTML, CSS, JS). Именно это нам и нужно.

Адрес сайта всегда строится так:
```
https://<твой-ник>.github.io/<название-репозитория>/
```

В нашем случае: `https://kotoroon.github.io/AnimeList/`

---

## Настройка (уже сделано)

1. Создали публичный репозиторий на GitHub
2. Загрузили туда файлы проекта (`index.html`, `manifest.json`, `sw.js`, иконки)
3. Включили GitHub Pages — указали ветку `master` и папку `/` (корень)

После этого GitHub сам собирает сайт и публикует его. Обычно занимает 1-2 минуты.

---

## Как обновить сайт после изменений

Вся магия в том, что GitHub Pages следит за веткой `master`. Как только ты пушишь новый коммит — сайт автоматически пересобирается.

### Шаги для обновления:

```bash
# 1. Перейди в папку проекта
cd ~/Development/AnimeList

# 2. Добавь все изменённые файлы
git add .

# 3. Сделай коммит с описанием что изменил
git commit -m "Описание изменений"

# 4. Отправь на GitHub
git push
```

Через 1-2 минуты изменения появятся на сайте.

### Проверить статус деплоя можно тут:
```
https://github.com/Kotoroon/AnimeList/actions
```

## Важные моменты

**Service Worker кэширует файлы** — после обновления сайта браузер на телефоне может ещё какое-то время показывать старую версию. Чтобы принудительно обновить — открой приложение и обнови страницу несколько раз, или очисти кэш браузера.

Если хочешь чтобы обновления применялись быстрее — при каждом обновлении меняй версию кэша в `sw.js`:
```js
const CACHE_NAME = "anime-tracker-v2"; // меняй число при каждом обновлении
```

**GitHub Pages работает только со статическими файлами** — никакого Node.js, Python и т.д. на сервере нет. Только то, что понимает браузер.
